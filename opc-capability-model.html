AI2C产品组 OPC 能力模型

用横向能力模型说明 INS、A1-A5、OPS、BIZ 的能力边界，并用 OPC-L1 到 OPC-L4 的圈层图定义产品组能力成熟度。

1. OPC 能力模型：横向拉伸

OPC 不是让 PM 变成所有岗位，而是让 PM 用 AI 扩大产品交付半径：左侧先把问题看准，右侧把用户和客户接住，中间把能力沉淀成可交付结果。

向左整合

01｜INS 洞察线

把散乱输入变成清晰问题：先看懂谁在什么场景里遇到什么痛点，再判断为什么现在要做、要做到什么程度。

INS调研摘要、痛点归因、场景链路、客群分层、目标定义、证据拆解。

PM Vibe Coding 能力阶梯 A1-A5

A1-A5 表示 PM 用 AI 完成从理解、推演、产出、交付到验证的 5 个阶段；能力深度按实际需求自然拉伸，不强行等分。

辅助理解

用 AI 做摘要、归纳、提问和场景拆解，能把业务输入整理成可讨论的问题。

方案推演

用 AI 生成初步方案，并通过多轮辩论判断方案的准确性、可执行性和业务适配度。

结构产出

用 AI 生成 PRD、流程、字段、规则和验收清单，让方案沉淀为可评审产物。

Demo 交付

用 AI 生成基础 Demo、还原业务场景，并推进到接近真实产品的前端形态。

上线验证与 Agent 修复闭环

用 AI 完成验收、静态部署、联调部署，并通过 Agent 修复体验问题，持续验证到可交付状态。

向右延伸

01｜OPS 2C 用户运营线

围绕上线后的用户目标，把需求拆成增长、体验、留存或收入动作，并能持续复盘。

OPS目标拆解、路径分析、触达策略、实验设计、数据复盘、迭代优化。

02｜BIZ 2B 商业转化线

产品经理能把产品能力转成客户听得懂的方案与价值表达，并直接对接客户推进试点、答疑和转化闭环。

BIZ客户诊断、方案包装、价值表达、演示答疑、试点推进、商务转化。
INS 洞察 ← 调研 / 痛点 / 场景 / 客群 / 目标 PM Vibe Coding 能力阶梯 A1-A5 OPS 2C 用户运营 + BIZ 2B 商业转化 → 直接对接客户

2. OPC 层级总览

OPC 层级是人的综合能力成熟度，能力坐标是具体技能点。层级越高，覆盖链路越长，单点能力越深。

OPC-L1｜AI 结构化执行者

让 AI 产物从“能看”变成“有结构、可评审”。

  • 掌握 AI 五件套。
  • 能做竞品、反馈、PRD 初稿、质量检查。
  • 产物包含缺口、风险、待确认问题。

OPC-L2｜AI 产品交付者

能独立交付一个可进入评审和基础验收的产品包。

  • 能拆场景、角色、对象、流程、状态。
  • 能输出 PRD、业务 Demo、评审材料、验收清单。
  • 能识别 AI 产物的事实错误和范围蔓延。

OPC-L3｜AI 工作流负责人

从个人交付，升级为团队可复用，并满足 A5-L2 静态部署能力。

  • 能沉淀 SOP、提示词模板、检查清单。
  • 能定义团队级产物 Rubric。
  • 能完成静态 Demo 部署，并组织他人按同一流程稳定产出。

OPC-L4｜AI 业务增长型 OPC

把 AI 能力用于跨部门交付和业务增长。

  • 能把客户问题转成方案、演示和价值表达。
  • 满足 A5-L3 联调部署与 Agent 修复，能直接参与客户沟通、试点和转化推进。
  • 能把用户数据和客户反馈反哺产品规划。
